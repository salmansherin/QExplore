System Settings
Listed below are the settings that have been chosen within config.inc.php, the config file for PHP Timeclock.
VARIABLE VALUE DESCRIPTION
MySql DB Settings
db_hostname: localhost This is the hostname for your mysql server, default is localhost.
db_name: timeclock This is the name of the mysql database you created during the install.
db_username: Sn_network This is the mysql username you created during the install.
db_password: ******** This is the mysql password for the username you created during the install.
db_prefix: This adds a prefix to the tablenames in the database. This can be helpful if you have an existing mysql database that you would like to use with PHP Timeclock. If you are unaware of what is meant by 'table prefix', then please leave this option as is. Default is to leave it blank.
dbversion: 1.4 This is the versioning number of the current database for PHP Timeclock.
 
Passwords and Security
use_passwd: yes This provides the option for the users to input their password when individually punching in/out of the timeclock. Default is "no".
use_reports_password: no If ALL users need access to ALL the reports provided, then set this to "no". Default is "no".
restrict_ips: no Choose "yes" to restrict the ip addresses that can connect to PHP Timeclock. If "yes" is chosen, you MUST input the allowed networks in the allowed_networks array below. Otherwise, choosing "yes" here and leaving allowed_networks blank will cause PHP Timeclock to reject everyone attempting to connect to it. Default is "no".
allowed_networks: view these via the 'Edit System Settings' link on the left. These are the networks or ip addresses you wish to allow to connect to PHP Timeclock. There is not a limit on how many networks or addresses that can be included in this array. This will currently only work for ipv4 addresses, ipv6 may be supported in a future release. If restrict_ips is set to "no", this option is ignored.
ip_logging: yes Enable the option to log the ip addresses of the connecting computers when users punch-in/out, or when a time is manually added, edited, or deleted. Default is "yes".
disable_sysedit: no Choosing "yes" disables ALL access to the Edit System Settings page (sysedit.php). It can be re-enabled in config.inc.php. Default is "no".
 
Dates and Times
Choose the way dates and times are to be displayed throughout the entire application:
calendar_style: amer Default is "amer".
datefmt: n/j/Y Default is "n/j/y".
js_datefmt: M/d/yyyy Default is "M/d/yy".
tmp_datefmt: m/d/yyyy Default is "m/d/yy".
timefmt: g:ia Default is "g:i a".
 
Reports Settings
Choose the way the reports are formatted by default. Most of these default settings can be changed when the reports are run.
round_time: 0 Choose how to round the time worked within the reports for each user. This simply tells the report how to format the total hours worked for the Hours Worked Report. Default is "0", meaning do not round.
paginate: yes Choose whether to paginate the Hours Worked report or not. Setting this option to "yes" will print each user's time on their own separate page. Default is "yes".
show_details: yes Choose whether to show the punch-in/out details for each punch for each user on the Hours Worked report or not. Default is "yes".
report_start_time: 00:00 These two variables, report_start_time and report_end_time, are designed to work with the Hours Worked report. They are there to provide a starting time to go along with the starting date, and an ending time to go along with the ending date for the dates specified when the report is run. Default is "00:00" (12:00am). 12 hour and 24 hour formats are supported.
report_end_time: 23:59 Default is "23:59" (11:59pm). 12 hour and 24 hour formats are supported.
username_dropdown_only: no Setting this variable to "yes" will display a single dropdown box containing usernames to choose from when running the reports. Setting this variable to "no" will instead display a triple dropdown box containing offices, groups, and usernames to choose from when running the reports. A single dropdown box works well if there are just a few usernames in the system, and a triple dropdown works well if multiple offices and/or groups are in the system. Default is "no".
user_or_display: user Choose whether to print displaynames or usernames for each user when reports are run. Options for this variable are "user" and "display". Default is "user".
display_ip: yes Choose whether to include in the reports the ip addresses of the systems that connect to sign-in/out into PHP Timeclock or not. This option is useful for auditing purposes. The ip_logging option must be set to "yes" in order for this option to work as expected. Default is "yes".
export_csv: no Choose "yes" to export the reports to a comma delimited file (.csv). Default is "no".
 
Timezone Settings
use_client_tz: no Setting this option to "yes" will display the punch-in/out times according to the timezone of the connecting computer, providing javascript is enabled in the user's browser. Default is "no".
use_server_tz: no Setting this option to "no" AND setting 'use_client_tz' to "no" will display the punch-in/out times according to your php.ini's date.timezone setting. If using PHP 5.1+ then choose "no" and set date.timezone in your php.ini file. Leaving date.timezone unset may cause PHP to choose the wrong timezone. Otherwise setting this option to "yes" will display the punch-in/out times according to the timezone of the web server through date("Z"). Default is "no".
 
Display Settings
color1: #EFEFEF When times are displayed in PHP Timeclock, they are displayed with these two alternating row colors. Default is "#EFEFEF".
color2: #FBFBFB Default is "#FBFBFB".
display_current_users: no Display only the current day's activity instead of the last entry from each user. Default is "no".
show_display_name: no Show a user's Display Name instead of their username on the main page. Default is "no".
display_office: all Display only a certain office on the main page of the application, instead of all the users. Default is "all".
display_group: all Display only a certain group on the main page of the application, instead of a particular office, or all the users. If "all" is chosen for the office, then you can choose any group in the list. This is there for if you have 2 or more groups with the same name, but with each having a different parent office. In this case, if you wanted to display all members of the groups with the same name, you could do this without having to choose an office. Default is "all".
display_office_name: no Display a column on the main page that shows the office each user is affiliated with. Default is "no".
display_group_name: no Display a column on the main page that shows the group each user is affiliated with. Default is "no".
display_weather: no To display local weather info on the left side of the application just below the submit button, set this to "yes". Default is "no".
weather_units: Celsius Display weather in US or metric measurements. Options are Fahrenheit or Celsius. Default is Celsius".
metar: CYYT Sets the ICAO (International Civil Aviation Organization) for your local airport. This is the unique four letter international ID for the airport. METAR reports are created at roughly 4500 airports from around the world, so you probably live near one of them. The airports make a report once or twice an hour, and these reports are stored at the National Weather Service and are publically available via HTTP or FTP. Visit https://pilotweb.nas.faa.gov/qryhtml/icao/ to find a corresponding ICAO near you. If 'display_weather' is set to "no", this option is ignored. If 'display_weather' is set to "yes", you MUST provide an ICAO here.
city: St. Johns, NL, Canada This is the city and state (or can be city and country) of the airport for the ICAO used above. If 'display_weather' is set to "no", this option is ignored.
links: none These are links to websites, other web-based applications, etc., that you wish to display in the topleft of the application just below the logo. Set it to "none" to ignore this option.
display_links: none These are the display names for the links chosen above. The number of display names MUST equal the number of links you have chosen above. If the 'links' option above is set to "none", this option is ignored.
logo: images/logos/phptimeclock.png This is a logo or graphic displayed in the top left of each page. Set it to "none" to ignore this option.
refresh: 300 Sets the refresh rate (in seconds) for the application. If PHP Timeclock is kept open, it will refresh this number of seconds to display the most current information. Set it to "none" to ignore this option. Default is 300.
 
Miscellaneous Settings
email: none This is an email address to display in the footer. Set it to "none" to ignore this option.
date_link: http://www.historychannel.com/tdih If your users click on the displayed date in the top right of the application, they will be taken to this website. Set it to "none" to ignore this option. Default is 'This Day in History', http://www.historychannel.com/tdih.
app_name: PHP Timeclock Sets the first half of the 'title'.
app_version: 1.04 Sets the second half of the 'title'.
title: PHP Timeclock 1.04 Sets the title in the header. This is what is displayed in the title bar of your web browser, and it is what the page will be named by default when you make a "favorite" or "bookmark" in your web browser.
